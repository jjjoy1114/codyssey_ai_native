제공해주신 파일들을 기반으로 깃허브(GitHub) 저장소(Repository)에 깔끔하게 올릴 수 있도록 파일 구조를 제안해 드리고, 저장소의 대문 역할을 할 **README.md** 파일의 내용을 작성해 드렸습니다.

깃허브에 올리실 때는 소상공인 정보 보호를 위해 실제 대화 로그에 포함된 전화번호나 개인 계정 정보(예: 메일 주소) 등은 가상의 정보(예: `010-0000-0000`, `user@email.com`)로 변경하여 업로드하시는 것을 추천합니다.

``` markdown
# 🤖 디지털 전환 가이드 챗봇 '다알려드리미' 프로젝트

본 프로젝트는 마케팅 인력과 시간이 부족하고 디지털 환경에 익숙하지 않은 자영업자 및 소상공인을 지원하기 위해 기획된 LLM 기반 디지털 전환 가이드 챗봇 **'다알려드리미'**의 개발 및 검증 기록입니다.

복잡한 IT 전문 용어를 배제하고 일상적인 언어를 사용하여 사용자의 심리적 장벽을 낮추고, 단계별 가이드를 통해 소상공인이 직접 홍보 영상을 제작하고 SNS 마케팅을 실행할 수 있도록 돕습니다.

## 🎯 프로젝트 개요
* **챗봇명:** 다알려드리미
* **기반 모델:** ChatGPT (무료 버전)
* **타겟 사용자:** 40~60대 자영업자 및 소상공인
* **핵심 가치:** 전문 용어 배제, 공감형 어조를 통한 심리적 안정감 제공, 즉각 실행 가능한 단계별 가이드
* **주요 검증 항목:** 환각 방지(존재하지 않는 기능 거부), 멀티턴 문맥 유지(업종 기억), 심리적 안정(실수 안심시키기)

## 📂 파일 구성 안내

### 1. [docs/](docs/) - 설계 및 보고서
* **[LLM 모델 비교·선정 보고서](docs/1_LLM_모델_비교_선정_보고서.md):** GPT, Claude, Gemini 등 다양한 모델을 비교 평가하여 소상공인 가이드에 최적화된 모델을 선정한 과정의 보고서입니다.
* **[시스템 설계 문서](docs/2_시스템_설계_문서.md):** 타겟 페르소나(김미영 사장님 사연 재구성) 정의, 프롬프트 실험 설계(간단 지시 vs 구체적 지시), 그리고 재사용성을 강화한 최종 GPTs 배포용 프롬프트 지시문이 포함되어 있습니다.

### 2. [logs/](logs/) - 대화 및 검증 로그

* **[실행 로그 및 검증 보고서](logs/3_실행_로그_및_검증_보고서.md):** 실제 대화 원본을 바탕으로 6턴(환각 방지), 9턴(문맥 유지), 10턴(심리적 안정) 등 핵심 검증 턴을 상세 분석하고 프롬프트 개선 이력을 정리한 보고서입니다.

## 🛠️ 핵심 성공 요인 (Key Insights)
1. **구체성 > 모델 종류:** 프롬프트의 구체성을 높이는 것이 모델의 체급을 바꾸는 것보다 어조 관리 및 용어 순화에 더 큰 성능 개선을 가져옴을 입증했습니다.
2. **사전 설계를 통한 위험 예방:** 발생한 오류를 사후에 수정하는 대신, 금지어 목록 지정 및 우선순위 규칙(심리적 안정 > 정보의 정확성) 설정을 통해 챗봇의 이탈을 사전에 차단했습니다.
3. **무료 환경 재현성 확보:** 유료 환경에서 검증된 우수한 성능이 무료 환경(ChatGPT Free)에서도 100% 동일하게 재현되어 소상공인의 실사용 가능성을 확보했습니다.
```
