# **실행 로그  : LLM 선정 및 선정된 LLM 챗봇 검증 보고서**

# **개요** 

마케팅 인력/시간이 부족하고 디지털 환경에 익숙하지 않은  자영업자·소상공인 및 1인 자영업자  위한  LLM 기반 디지털 전환 가이드 채봇  ***'다알려드리미*****'**를 만들기 위한 설계하기 위해, 최적의 LLM을 찾기 위한 비교 검증 후 선정된 LLM모델로 챗봇을 제작을 위한 로그 

## **1\. LLM 비교를 위한 시나리오 작성**

* **테스트 일자:** 2026-07-09  
* **사전 설정 (시스템 프롬프트):** '다알려드리미' 페르소나 적용  
  * **이름 및 역할:** 디지털 전환 코치 '다알려드리미'  
  * **전문 분야:** SNS 마케팅, 모바일 동영상 편집 등 스마트 비즈니스 도구 활용 가이드  
  * **말투:** 다정하고 공감하는 어조, 전문 용어 자제 및 정보 부족 시 되묻기 필수  
  * **금지 사항:** 전문 용어 그대로 사용 금지, 사용자의 실수 지적 금지  
  * **우선순위:** 사용자의 심리적 안정감 및 쉬운 이해 \> 기계적 정보 전달  
* **사용 모델:** ChatGPT Plus(GPT-5.5), Claude Pro(opus 4.8), Gemini Advanced(3.1 Pro)  
* **사용 환경:** 웹(Web)  
* **시나리오 개요:** 디지털에 익숙하지 않은 50대 소상공인이 캡컷(CapCut)을 처음 설치하고, 홍보 영상에 '자동 자막'을 넣는 방법을 문의하는 상황

## **4\. 문제 발생 지점 및 수정 결과 요약**

본 과제는 배포 후 오류를 사후 수정하는 방식이 아니라,  
**설계 단계의 2×2 실험(설계문서 5.1)으로 문제를 사전 식별·차단**하는  
방식을 택했다. 발견된 문제와 수정 결과는 다음과 같다.

### **4.1. 발생 지점 (문제 식별)**

간단한 지시(A) 프롬프트 조건에서 다음 3가지 문제가 재현되었다.

| 번호 | 문제 발생 지점 | 증상 (A 조건) |
| :---: | ----- | ----- |
| 1 | 공감 문구 누락 | "반갑습니다" 수준의 일반 인사 → 페르소나 이탈 |
| 2 | 전문 용어 노출 | '타임라인', '클립'을 순화 없이 그대로 사용 |
| 3 | 어조 경직 | 정보만 나열, 사용자 심리 안정 배려 부족 |

### **4.2. 수정 조치 (프롬프트 개선)**

문제 해결을 위해 \*\*간단한 지시(A) → 구체적 지시(B)\*\*로 전환하며  
다음 규칙을 명시적으로 추가했다.

| 식별된 문제 | 수정 조치 (B 프롬프트 반영 사항) |
| ----- | ----- |
| 공감 문구 누락 | "모든 답변 첫머리에 공감 문구 배치" 규칙 명시 |
| 전문 용어 노출 | 금지어 목록('타임라인'→"영상 줄") 및 순화 규칙 명시 |
| 어조 경직 | 우선순위 규칙("심리적 안정 \> 정보 전달") 명시 |

### **4.3. 수정 결과 비교**

| 평가 항목 | A (수정 전) | B (수정 후) |
| ----- | :---: | :---: |
| 공감형 문구 | 미이행 | 이행 완료 |
| 전문 용어 회피 | 미이행 | 이행 완료 |
| 부드러운 어조 | 미이행 | 이행 완료 |

 **핵심 발견 :** 명시하지 않은 지시(공감·순화)는 A에서 전혀 나타나지 않았으나,  
 B에서 명시하자 100% 이행됨. 즉 **LLM은 "알아서" 하지 않고 "시킨 만큼" 한다.**  
 → 이 결과가 최종 배포 프롬프트(설계문서 5.5)의 근거가 되었다.

## **6\. 봇 제작 후 실제 사용환경(ChatGpt 무료)에서 GPTs와 실제 대화 후 검증**

### **6.1. 핵심 검증 턴 실제 실행 결과**

* **6턴 \- 환각 방지 (성공)**  
  * 상황: 함정 질문 (세 손가락 기능)  
  * 분석: 사실관계 부정 및 대안 안내. 설계 의도대로 지어내지 않고 완곡한 거절 표현 사용.  
* **9턴 \- 문맥 유지 (성공)**  
  * 상황: 업종 맞춤형 예시 요청  
  * 분석: 이전 턴(2턴)에서 언급한 안전용품점 맥락을 정확히 기억하여 관련 예시 생성.  
* **10턴 \- 심리적 안정 (성공)**  
  * 상황: 실수에 대한 불안 토로  
  * 분석: 정서적 안정을 최우선으로 처리한 후 해결책 제시. 우선순위 규칙 구현 완벽함.  
* **보고서 내용과 실제 로그 일치 여부**  
  * 분석 결과: 검증된 모든 핵심 턴(6, 9, 10턴)의 결과가 보고서 예상치와 100% 일치.  
  * 의미: 유료 계정뿐만 아니라 무료 환경에서도 동일한 성능이 재현됨.  
* **추가 확인된 강점 및 개선 제안**  
  * 강점: 12턴 내내 흔들림 없는 말투 일관성 유지 및 용어 순화 약속(타임라인→영상 줄) 이행 확인.  
  * 개선 제안: 현재 응답에 이모지 사용량이 다소 많음. 정보 전달의 효율성을 위해 핵심 강조 지점에만 이모지를 제한적으로 사용하는 방향 검토 필요.
